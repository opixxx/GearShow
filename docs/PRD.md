# GearShow PRD (Product Requirements Document)

---

## 1. 프로젝트 개요

GearShow는 축구 장비(축구화, 유니폼 등)를 **3D 모델 기반으로 시각화**하고, 사용자 경험 데이터를 함께 제공하는 **쇼케이스 플랫폼**이다.

사용자는 자신이 소유한 장비의 상태, 착용 후기, 실물 사진을 등록하고, 이를 3D 모델로 변환하여 다른 사용자에게 공유할 수 있다. 또한 사용자 간 직거래 및 안전거래를 통해 장비를 거래할 수 있다.

---

## 2. 문제 정의 (Problem Statement)

### 2-1. 현재 시장의 문제

- **정보 비대칭**: 중고 축구 장비를 구매할 때 실물 상태를 정확히 파악하기 어렵다. 평면 사진만으로는 장비의 실제 컨디션(밑창 마모, 스터드 상태, 원단 손상 등)을 판단하기 어렵다.
- **분산된 거래 환경**: 축구 장비 거래가 카페, 중고 플랫폼, SNS 등 여러 곳에 분산되어 있어 원하는 모델을 찾기 어렵고, 장비에 특화된 정보(스터드 타입, 사일로, 시즌 등)가 체계적으로 관리되지 않는다.
- **경험 데이터 부재**: 장비 구매 시 다른 사용자의 실착 후기, 사이즈 정보, 착용 횟수 등 경험 기반 데이터를 체계적으로 확인할 수 없다.

### 2-2. GearShow의 해결 방안

| 문제 | 해결 방안 |
|:-----|:---------|
| 실물 상태 파악 어려움 | 3D 모델 기반 시각화로 장비를 360도 확인 가능 |
| 분산된 거래 환경 | 축구 장비 전문 카탈로그 + 쇼케이스 + 거래 통합 플랫폼 |
| 경험 데이터 부재 | 상태 등급, 착용 횟수, 사이즈 정보 등 구조화된 경험 데이터 제공 |

## 3. 타겟 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|:-----------|:-----|:---------|
| 축구 장비 수집가 | 다양한 축구화/유니폼을 수집하고 공유하고 싶은 사용자 | 3D 쇼케이스, 컬렉션 공유 |
| 중고 장비 판매자 | 사용하지 않는 장비를 판매하고 싶은 사용자 | 신뢰 기반 거래, 실물 상태 전달 |
| 중고 장비 구매자 | 합리적인 가격으로 장비를 구매하고 싶은 사용자 | 정확한 상태 확인, 실착 후기 |
| 일반 축구 동호인 | 장비 정보를 탐색하고 후기를 참고하고 싶은 사용자 | 카탈로그 검색, 경험 데이터 열람 |

## 4. 핵심 기능

### 4-1. 쇼케이스 (Showcase)

플랫폼의 **핵심 기능**이다. 사용자가 소유한 장비를 등록하고 공유한다.

- 카탈로그 아이템 기반 장비 등록
- 상태 등급(S/A/B/C), 착용 횟수, 사이즈 등 경험 데이터 입력
- 실물 사진 업로드 (최소 1장, 대표 이미지 1개 필수)
- 댓글을 통한 사용자 간 소통

### 4-2. 3D 모델 (3D Model)

쇼케이스의 **차별화 기능**이다. 실물 사진을 3D 모델로 변환한다.

- 4방향(앞/뒤/좌/우) 소스 이미지 기반 3D 모델 자동 생성
- 외부 3D 생성 서비스(Tripo 등) 연동
- 생성 상태 추적 (요청 → 생성 중 → 완료/실패)
- 3D 뷰어를 통한 360도 장비 확인

### 4-3. 카탈로그 (Catalog)

플랫폼에 등록된 **공식 장비 정보 데이터베이스**이다.

- 축구화: 브랜드, 모델명, 스터드 타입, 사일로, 출시 연도 등
- 유니폼: 클럽명, 시즌, 리그, 제조사 등
- 쇼케이스 등록 시 카탈로그 아이템과 연결

### 4-4. 사용자 (User)

소셜 로그인 기반 인증 및 사용자 관리이다.

- 소셜 로그인 (Kakao, Google, Apple)
- 프로필 관리 (닉네임, 프로필 이미지)
- 휴대폰 인증 (거래 기능 사용 시 필수)

### 4-5. 거래 (Transaction)

사용자 간 장비 거래를 지원한다.

- 직거래: 채팅 협의 후 오프라인 거래, 판매자가 판매 완료 확정
- 안전거래: 플랫폼 중개, 결제 → 발송 → 수령 확인 → 정산

### 4-6. 채팅 (Chat)

거래를 위한 1:1 실시간 채팅이다.

- 쇼케이스 기반 판매자-구매자 1:1 채팅방
- 텍스트, 이미지, 시스템 메시지 지원

## 5. 개발 우선순위

### Phase 1 — MVP 핵심 (현재 진행 중)

> 목표: 3D 쇼케이스 등록/조회가 가능한 최소 제품

| 순서 | 도메인 | 범위 | 사유 |
|:--:|:------|:-----|:-----|
| 1 | **User + Auth** | 소셜 로그인, 프로필 관리 | 모든 기능의 전제 조건, 인증 없이 쇼케이스 등록 불가 |
| 2 | **Catalog** | 카탈로그 아이템 등록/조회 | Showcase 등록 시 카탈로그 아이템 선택 필수 |
| 3 | **Showcase + 3D Model** | 쇼케이스 CRUD, 3D 모델 생성/조회, 이미지 업로드, 댓글 | 플랫폼 핵심 기능, User + Catalog 완성 후 개발 |

### Phase 2 — 거래 기능

> 목표: 사용자 간 장비 거래가 가능한 플랫폼

| 순서 | 도메인 | 범위 | 사유 |
|:--:|:------|:-----|:-----|
| 4 | **Chat** | 1:1 채팅방, 메시지 송수신 | 거래 협의를 위한 채팅 필요 |
| 5 | **Transaction** | 직거래/안전거래, 거래 상태 관리 | 채팅 기반 거래 흐름 |
| 6 | **Payment** | 결제 연동 (Toss, Kakao) | 안전거래 시 결제 필요 |

### Phase 3 — 고도화

> 목표: 사용자 경험 개선 및 플랫폼 확장

- 검색/필터 고도화
- 알림 시스템 (거래 상태 변경, 댓글 알림 등)
- 리뷰/평점 시스템
- 관리자 대시보드

## 7. 범위 정의 (Scope)

### In Scope (포함)

- 소셜 로그인 기반 사용자 인증
- 카탈로그 아이템 관리
- 쇼케이스 CRUD + 이미지 관리
- 3D 모델 생성 및 조회
- 댓글 기능
- 1:1 채팅
- 직거래 / 안전거래
- 결제 연동

### Out of Scope (제외)

- 관리자용 백오피스 (Phase 3 이후 검토)
- 실시간 알림/푸시 (Phase 3 이후 검토)
- AI 기반 가격 추천
- 배송 추적 연동

## 8. 성공 지표 (KPI)

| 지표 | 설명 | 목표 (MVP) |
|:----|:-----|:----------|
| 쇼케이스 등록 수 | 사용자가 등록한 쇼케이스 총 수 | - |
| 3D 모델 생성 성공률 | 요청 대비 COMPLETED 비율 | 80% 이상 |
| 3D 모델 생성 소요 시간 | 요청 → 완료까지 평균 시간 | 5분 이내 |
| DAU (일간 활성 사용자) | 하루 동안 플랫폼을 이용한 사용자 수 | - |
| 거래 전환율 | 쇼케이스 조회 → 거래 요청 비율 | - |

> MVP 단계에서는 정량적 목표보다 **기능 동작 검증**에 집중한다.
