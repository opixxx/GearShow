Presigned URL 업로드 전환 설계

GearShow — LocalStack(서버 경유) → AWS S3(Presigned URL 직접 업로드)

현재 구조 vs 변경 후 구조

현재: 서버 경유 업로드

  • 이미지가 서버 메모리를 거쳐 S3로 업로드
  • 10MB 파일 제한, 서버 대역폭 소모
  • 동시 업로드 시 서버 메모리 압박
  • Flutter → multipart/form-data → Spring → S3
  • 서버가 단일 실패점 (SPOF)

변경: Presigned URL 직접 업로드

  • 이미지가 서버를 거치지 않고 S3에 직접 업로드
  • 서버 메모리/대역폭 부담 제거
  • 대용량 파일도 안정적 처리
  • Flutter → S3 직접 PUT (서버는 URL 발급만)
  • S3의 높은 가용성 활용
쇼케이스 등록 전체 플로우
Flutter 앱에서 쇼케이스를 등록할 때의 전체 흐름 (이미지 + 3D 모델 소스 이미지)
1
Flutter (Client)
Presigned URL 발급 요청
업로드할 이미지 개수와 파일 정보(Content-Type, 파일명)를 서버에 전달한다.
POST/api/v1/showcases/upload-urls
{
  "files": [
    { "contentType": "image/jpeg", "filename": "photo1.jpg", "type": "SHOWCASE_IMAGE" },
    { "contentType": "image/jpeg", "filename": "photo2.jpg", "type": "SHOWCASE_IMAGE" },
    { "contentType": "image/jpeg", "filename": "front.jpg", "type": "MODEL_SOURCE" }
  ]
}
2
Spring Boot (Server)
Presigned URL 생성 및 반환
각 파일에 대해 UUID 기반 S3 key를 생성하고, S3Presigner로 PUT용 Presigned URL을 발급한다.
만료 시간: 10분, Content-Type 고정.
{
  "uploadUrls": [
    {
      "presignedUrl": "https://s3.../showcases/uuid1.jpg?X-Amz-...",
      "s3Key": "showcases/uuid1.jpg",
      "type": "SHOWCASE_IMAGE"
    },
    { ... },
    { ... }
  ]
}
3
Flutter (Client)
S3에 직접 이미지 업로드
발급받은 Presigned URL로 각 이미지를 S3에 직접 PUT 요청한다.
서버를 거치지 않으므로 서버 부하 없음. 병렬 업로드 가능.
PUThttps://s3.ap-northeast-2.amazonaws.com/...?X-Amz-Signature=...
Headers: { "Content-Type": "image/jpeg" }
Body: [이미지 바이너리 데이터]
4
AWS S3
이미지 저장 완료
S3가 파일을 저장하고 200 OK를 반환한다.
이미지는 CDN URL로 접근 가능: https://cdn.gearshow.com/showcases/uuid1.jpg
5
Flutter (Client)
쇼케이스 생성 요청 (업로드 완료 확인)
모든 이미지 업로드가 완료되면, S3 key 목록과 쇼케이스 정보를 서버에 전달한다.
더 이상 multipart/form-data가 아닌 JSON 요청.
POST/api/v1/showcases
{
  "title": "머큐리얼 슈퍼플라이 10 엘리트",
  "category": "BOOTS",
  "brand": "Nike",
  "conditionGrade": "A",
  "imageKeys": ["showcases/uuid1.jpg", "showcases/uuid2.jpg"],
  "primaryImageIndex": 0,
  "modelSourceImageKeys": ["showcases/uuid3.jpg"],
  "spec": { "studType": "FG", ... }
}
6
Spring Boot (Server)
S3 key 검증 및 쇼케이스 저장
전달받은 S3 key가 실제로 S3에 존재하는지 headObject로 검증한다.
CDN URL을 조합하여 ShowcaseImage 엔티티를 생성하고 DB에 저장한다.
3D 모델 소스 이미지가 있으면 비동기로 3D 모델 생성을 요청한다.

이미 등록된 쇼케이스에 이미지를 추가할 때

API 변경 요약
상태 Method Endpoint 설명
NEW POST /api/v1/showcases/upload-urls 쇼케이스 등록용 Presigned URL 발급
NEW POST /api/v1/showcases/{id}/images/upload-urls 이미지 추가용 Presigned URL 발급
MODIFY POST /api/v1/showcases multipart → JSON 변경 (S3 key 전달)
MODIFY POST /api/v1/showcases/{id}/images multipart → JSON 변경 (S3 key 전달)
MODIFY POST /api/v1/showcases/{id}/3d-model multipart → JSON 변경 (S3 key 전달)
KEEP DELETE /api/v1/showcases/{id}/images/{imgId} 변경 없음 (서버에서 S3 삭제)
KEEP PATCH /api/v1/showcases/{id}/images/order 변경 없음
보안 설정
!
Presigned URL 설정
보안 제약 조건
만료 시간: 10분 (업로드 충분, 악용 방지)
Content-Type 고정: image/jpeg, image/png, image/webp만 허용
파일 크기: S3 버킷 정책으로 10MB 제한
인증 필수: Presigned URL 발급 API는 로그인 사용자만 접근 가능
S3 key 검증: 쇼케이스 생성 시 headObject로 파일 존재 여부 확인
!
S3 버킷 정책
버킷 보안 설정
Public Access: 차단 (Block all public access)
IAM 권한: 서버에 s3:PutObject, s3:GetObject, s3:DeleteObject만 부여
CDN 연동: CloudFront를 통해서만 읽기 접근 허용 (OAI/OAC)
CORS: Flutter 웹에서 직접 PUT 가능하도록 S3 CORS 설정 필요
백엔드 변경 범위
계층 파일 변경
Infrastructure S3Config.java MODIFY S3Presigner 빈 추가
Port ImageStoragePort.java MODIFY generatePresignedUrl(), verifyExists() 메서드 추가
Adapter S3ImageStorageAdapter.java MODIFY Presigner 구현, headObject 검증 구현
Application UploadFile.java DELETE InputStream 불필요 (서버가 파일을 안 받으므로)
Adapter UploadFileMapper.java DELETE MultipartFile 변환 불필요
Application CreateShowcaseFacade.java MODIFY S3 업로드 단계 제거, S3 key 검증으로 변경
Controller ShowcaseController.java MODIFY multipart → JSON, URL 발급 엔드포인트 추가
Controller ShowcaseImageController.java MODIFY multipart → JSON, URL 발급 엔드포인트 추가
Controller ShowcaseModel3dController.java MODIFY multipart → JSON 변경
DTO CreateShowcaseRequest.java MODIFY imageKeys, modelSourceImageKeys 필드로 변경
S3 Key 경로 전략
showcases/{uuid}.{ext} — 쇼케이스 이미지
showcases/model-source/{uuid}.{ext} — 3D 모델 소스 이미지
CDN URL 조합: https://cdn.gearshow.com/{s3Key}

변경 필요
이미지 업로드 로직 변경

  2) 각 URL로 S3에 직접 PUT 업로드 (병렬 가능)
  3) S3 key 목록을 서버에 전달하여 쇼케이스 생성
