# 위험도 등급과 머지 정책

모든 변경은 EXEC_PLAN의 **Risk** 필드에 등급을 선언한다.
등급에 따라 자동 머지 허용 범위와 필요 승인 절차가 달라진다.

## 등급표

| 변경 유형 | 위험도 | 자동 머지 | 사람 리뷰 | 추가 요구사항 |
|---|---|---|---|---|
| 오타·포맷·주석만 | 🟢 Safe | ✅ 허용 | 선택 | — |
| 테스트 코드 추가 | 🟢 Safe | ✅ 허용 | 선택 | — |
| 내부 리팩토링 (공개 API 불변) | 🟡 Caution | 🟡 서브에이전트 Critical 0일 때만 | 권장 | before/after 동작 동일성 증빙 |
| 새 API / 새 유스케이스 | 🟠 Risky | ❌ | **필수** | API 명세 문서 갱신 |
| 공개 API Breaking Change | 🔴 High | ❌ | **필수** | 마이그레이션 가이드 |
| DB 스키마 변경 | 🔴 High | ❌ | **필수** | **롤백 스크립트 첨부** |
| 상태기계 변경 (3D 생성, 주문 등) | 🔴 High | ❌ | **필수** | **ADR 작성** |
| Kafka 이벤트 계약 변경 | 🔴 High | ❌ | **필수** | 스펙 문서 갱신 + 컨슈머 영향도 |
| 보안 경계 변경 (인증·인가·PII) | 🔴 High | ❌ | **필수** | 보안 리뷰 |

## 판정 원칙

- 확신이 없으면 **한 등급 위로**. 🟡 vs 🟠 사이면 🟠.
- 한 PR에 여러 변경이 섞이면 **가장 높은 등급**을 따름.
- 🔴 항목은 반드시 별도 PR로 분리 — 다른 변경에 묻히지 않게.

## 커밋 메시지 형식

한글 "타입: 요약" 형식. 에스컬레이션 필요 변경은 접두사에 표시.

```
feat: 쇼케이스 3D 생성 실패 알림 이벤트 추가       # 🟠
fix: REQUESTED→PREPARING 전환 원자성 보장         # 🟢 (버그 수정)
refactor!: Showcase.create()에 options 인자 추가  # 🔴 (Breaking, !표시)
chore: ArchUnit 규칙 추가                         # 🟢
```
